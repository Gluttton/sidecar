Detailed Description

Helper class for Controller objects that do not spawn a thread for algorithm message processing (see Controller::openAndInit()).

The story here is that we will receive notify() calls from a Controller's message queue when messages are added. We then notify our ACE reactor which will schedule a call to our handle_input() call when we are in the main thread, which in turn calls on Controller::fetchAndProcessOneMessage() to process the message in the queue.
