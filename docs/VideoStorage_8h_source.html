11 //VideoStorage is a simple deque encapsulation used to store all video
12 // data. Video data must be stored until the longest (in pri counts)
13 // target has been released (i.e., closed). When the target is closed,
14 // the code will request the window of video data that the target overlaps
15 // and merge the two for centroiding. Note that we cannot know what video
16 // data a target will need before it is closed. This is because we want
17 // all video data, even where the mask is not defined and because target
18 // regions can merge in unpredictable ways. when a target is released,
19 // the calling code will recalculate how many video pri's need to be stored
20 // and then call SetDepth. SetDepth must always be less than the current
21 // depth. any pri's greater than the new depth will be released (so that
22 // we don't end up storing the video history since the beginning of time).
