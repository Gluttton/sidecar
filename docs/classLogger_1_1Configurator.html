Detailed Description

Logger configuration manager class.

A configuration is defined by the contents of a config file (or stream), where each line specifies a Log object and its setup. The format of a config line is:

NAME [priority P] [[writer W [W_ARGS] formatter F [F_ARGS]] ...]

where NAME is the name of the Log object (including lineage); P is the priority level the Log object will accept messages; W is the name of a Writer object to assign to the Log object; W_ARGS are any optional settings for the Writer constructor; F is the name of the Formatter object to use for the Writer; and F_ARGS are any optional settings for the Formatter constructor.

The priority and writer constructs are optional; there may be either or both on the same line, or they may appear on separate config lines. There may even be more than one writer specification on the same line, or each one may appear on a separate config line. In short, all settings are additive and all config lines are processed in the order in which they appear in a file or stream.

The following are valid W values for the writer keyword:

cout

Creates a new StreamWriter object, writing to the std::cout stream. The only option available to this (and all other) StreamWriter object is

[flush]

which controls whether the stream's flush method is called after each log message write.

clog

Same as above except the StreamWriter object uses std::clog for its output stream

cerr

Same as above except the StreamWriter object uses std::cerr for its output stream

file

Creates a new FileWriter object. The arguments to a FileWriter are:

PATH [nappend | mode M]

where PATH specifies a file used to hold log messages (required), and M is an optional mode specification used for newly-created files (see open(2))

If nappend is found, then FileWriter will overwrite any existing log file; the default behavior is to append to the end of an existing file.

rolling

Create a new RollingWriter object. The arguments to a RollingWriter are:

PATH [nappend | mode M | versions V | size S]

where PATH, M, and nappend are the same as for FileWriter; V is the number of versions of the log file to keep around; S is the maximum size of the log file – if the log file reaches this size, then the file is versioned, and then a new log file is created.

syslog

Create a new SyslogWriter object which sends all log messages to a syslog daemon (see syslog(3C)). The arguments to a SyslogWriter are:

IDENT [facility F]

where IDENT is the required identification string given to openlog and found in all log messages generated by syslog. F is an optional facility specification as defined by the syslog implementation.

remote

Create a new RemoteSyslogWriter object which will send all log messages over the network as datagrams to a remote syslog server. The arguments to a RemoteSyslogWriter are:

HOST [port P]

where HOST is the required host name or IP address of the remote host to connect to, and P is an optional port number (default is 514).

The following are valid F values for the formatter keyword:

terse
Use a TerseFormatter object for log message formatting.
verbose
Use a VerboseFormatter object for log message formatting.
pattern
Use a PatternFormatter object for log message formatting. The sole argument to a PatternFormatter is a string that specifies how to format a log message.
