Detailed Description

Records statistics for all unit tests run.

If any unit test throws an exception, we keep a copy of it. There are two types of exceptions covered: failures (represented by TestException exceptions) and errors (caused by anything else). The former represents a unit test that failed to pass, whereas the latter represents a problem running the test.
