Detailed Description

Interface and baseline functionality for all unit test objects.

Can be used as a simple unit test by subclassing and overriding TestObj::test() method. Here's a simple check to make sure that 1 is really one.

struct Foo : public TestObj
{
Foo() : one_(1) {}
void test() { assertEqual(1, one_); }
int one_;
};
int
main()
{
return Foo().mainRun(); // run the above test
}

Please note that assertEqual above is not a member of TestObj or of Foo. Rather, it is a C preprocessor function macro found in TestObj.h. Although this is a gross violation of standard coding practices, these C preprocessor function macros allow a unit test to record the file and line number where the test was if it failed. They do so by using the values of the FILE and LINE C preprocessor value macros as arguments to the TestObj methods they invoke (the alternative would be to have the developer type in FILE and LINE for each invocation of _assertEqual() or its kin)

See TestObj Function Macros for a documentation on these function macros.
