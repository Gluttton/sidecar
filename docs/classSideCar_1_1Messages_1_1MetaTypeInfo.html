Detailed Description

Unique identification for message types/classes.

Each unique message class must have an entry in the MetaTypeInfo::Value enum. It is primarily used to locate message loading procedures that unmarshal data read in from an external data source.

enum SideCar::Messages::MetaTypeInfo::Value : uint16_t
strong

Active message types in the SideCar system.

The ordering is not important, though kInvalid should always be first, and kUnassigned should always be last.

NOTE: if you add to this, add to the END of the list, before kUnassigned. Otherwise, past recordings won't play correctly.
